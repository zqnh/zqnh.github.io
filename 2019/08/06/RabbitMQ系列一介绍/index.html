RabbitMQ系列一介绍

发表于 2019-08-06 | 分类于 RabbitMQ | |
消息中间件RabbitMQ

RabbitMQ是使用Erlang语言开发的开源消息队列系统，基于AMQP协议来实现。AMQP的主要特征是向消息、队列、路由(包括点对点和发布/订阅)、可靠性安全。AMQP协议更多用在企业系统内，对数据一致性、稳定性和可靠性要求很高的场景，对性能和吞吐量的要求还在其次。

RabbitMQ核心概念以及AMQP协议

AMQP高级消息队列协议

AMQP：Advanced Message Queuing Protocol

定义：是具有现代特征的二进制协议。是一个提供消息服务的应用层标准高级消息队列协议，是应用层协议的一个开放标准，为面向消息的中间件设计。

核心概念:

:v:Server:又称Broker，接受客户端的连接，实现AMQP实体服务；

:v:Connection：连接，应用程序与Broker的网络连接；

:v:Channel:网络信道，几乎所有的操作都在Channel中进行，Channel是哪个消息读写的通道。客户端可建立多个Channel，每个Channel代表一个会话任务。

:v:Message：消息，服务器和应用程序之间传送的数据，又Properties和Body组成。Properties可以对消息进行修饰，比如消息的优先级、延迟等高级特性；Body则就是消息体内容。

:v: Virtual host: 虚拟地址，用于进行逻辑隔离，最上层的消息路由，一个Virtual Host里面可以有若干个Exchange和Queue，同一个Virtual Host里面不能有相同名称的Exchange或Queue​

:v:Exchange：交换机，接受消息，根据路由键转发消息到绑定的队列

:v: Binding：Exchange和Queue之间的虚拟连接，binding中可以包含routing key

:v: Routing key：一个路由规则，虚拟机可用它来确定如何路由一个特定消息

:v: Queue：也称为Message Queue，消息队列，保存消息并将它们转发给消费者
